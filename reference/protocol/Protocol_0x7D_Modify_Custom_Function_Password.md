### 0x7d Modify Password of Tag Customised Functions Command

이 명령어는 태그의 커스텀 기능(예: Monza 4QT의 'Peek' 기능)에 접근할 때 사용되는 암호를 수정하는 데 사용됩니다. 이 설정은 리더기 전원이 꺼져도 유지됩니다.

#### 1. 요청 프레임 (PC → E310)

**프레임 구조:**

| Len | Adr | Cmd | Data[] | CRC-16 (LSB) | CRC-16 (MSB) |
| :-- | :-- | :-- | :-- | :--- | :--- |
| 8 | 1 | 1 | 4 | 1 | 1 |

**필드 설명:**

| 필드 | 길이 (Byte) | 설명 |
| :--- | :--- | :--- |
| `Len` | 1 | `0x08`. |
| `Adr` | 1 | 리더기 주소 (0x00 ~ 0xFE). 0xFF는 브로드캐스트. |
| `Cmd` | 1 | 명령어 코드. `0x7d`로 고정. |
| `Data[]`| 4 | 설정할 새로운 커스텀 기능 접근 암호. 아래 "Data[] 필드 포맷" 참조. |
| `CRC-16`| 2 | `Len`부터 `Data[]`까지의 CRC-16 체크섬. |

**Data[] 필드 포맷:**

| UserPwd |
| :--- |
| 4 |

**Data[] 파라미터 설명:**

| 파라미터 | 길이 (Byte) | 설명 |
| :--- | :--- | :--- |
| `UserPwd`| 4 | 사용자가 정의하는 새로운 4바이트 접근 암호. 상위 바이트(Most-significant byte)가 먼저 전송됩니다. |

---

#### 2. 응답 프레임 (E310 → PC)

**프레임 구조:**

| Len | Adr | reCmd | Status | Data[] | CRC-16 (LSB) | CRC-16 (MSB) |
| :-- | :-- | :--- | :--- | :--- | :--- | :--- |
| 5 | 1 | 1 | 1 | 0 | 1 | 1 |

**필드 설명:**

| 필드 | 길이 (Byte) | 설명 |
| :--- | :--- | :--- |
| `Len` | 1 | `0x05`. |
| `Adr` | 1 | 리더기 주소. |
| `reCmd` | 1 | 응답하는 명령어 코드. `0x7d`로 고정. |
| `Status`| 1 | 명령어 실행 상태. 성공 시 `0x00`. |
| `Data[]`| 0 | 데이터 없음. |
| `CRC-16`| 2 | `Len`부터 `Status`까지의 CRC-16 체크섬. |
