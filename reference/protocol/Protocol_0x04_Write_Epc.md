### 0x04 Write EPC Number Command

`Write EPC Number` 명령어는 태그의 EPC 번호를 새로 쓰는 데 사용됩니다. 이 작업을 수행하는 동안 안테나 유효 범위 내에는 하나의 태그만 있어야 합니다.

#### 1. 요청 프레임 (PC → E310)

**프레임 구조:**

| Len | Adr | Cmd | Data[] | CRC-16 (LSB) | CRC-16 (MSB) |
| :-- | :-- | :-- | :-- | :--- | :--- |
| 가변 | 1 | 1 | 가변 | 1 | 1 |

**필드 설명:**

| 필드 | 길이 (Byte) | 설명 |
| :--- | :--- | :--- |
| `Len` | 1 | `Adr`부터 `CRC-16` MSB까지의 총 바이트 수. |
| `Adr` | 1 | 리더기 주소 (0x00 ~ 0xFE). 0xFF는 브로드캐스트. |
| `Cmd` | 1 | 명령어 코드. `0x04`로 고정. |
| `Data[]`| 가변 | EPC 쓰기 작업에 필요한 파라미터. 아래 "Data[] 필드 포맷" 참조. |
| `CRC-16`| 2 | `Len`부터 `Data[]`까지의 CRC-16 체크섬. |

**Data[] 필드 포맷:**

| ENum | Pwd | WEPC |
| :--- | :-- | :--- |
| 1 | 4 | 가변 |

**Data[] 파라미터 설명:**

| 파라미터 | 길이 (Byte) | 설명 |
| :--- | :--- | :--- |
| `ENum` | 1 | 새로 쓸 EPC 번호의 길이를 워드(word) 단위로 지정. (0 <= `ENum` < 15) |
| `Pwd` | 4 | 태그 접근 암호. |
| `WEPC` | 가변 | 태그에 새로 쓸 EPC 데이터. `ENum`으로 지정된 워드 수와 동일해야 함. (1 <= length <= 31 words) |

---

#### 2. 응답 프레임 (E310 → PC)

**프레임 구조:**

| Len | Adr | reCmd | Status | Data[] | CRC-16 (LSB) | CRC-16 (MSB) |
| :-- | :-- | :--- | :--- | :--- | :--- | :--- |
| 5 | 1 | 1 | 1 | 0 | 1 | 1 |

**필드 설명:**

| 필드 | 길이 (Byte) | 설명 |
| :--- | :--- | :--- |
| `Len` | 1 | `0x05`. |
| `Adr` | 1 | 리더기 주소. |
| `reCmd` | 1 | 응답하는 명령어 코드. `0x04`로 고정. |
| `Status`| 1 | 명령어 실행 상태. 성공 시 `0x00`. (자세한 내용은 매뉴얼 5장 참조) |
| `Data[]`| 0 | 데이터 없음. |
| `CRC-16`| 2 | `Len`부터 `Status`까지의 CRC-16 체크섬. |
