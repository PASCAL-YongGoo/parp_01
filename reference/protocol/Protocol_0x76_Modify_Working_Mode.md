### 0x76 Modify Working Mode Command

이 명령어는 리더기의 전체적인 동작 모드를 변경하는 데 사용됩니다. 이 설정은 리더기 전원이 꺼져도 유지되며, 재시작 시 이전에 설정된 모드로 동작합니다.

#### 1. 요청 프레임 (PC → E310)

**프레임 구조:**

| Len | Adr | Cmd | Data[] | CRC-16 (LSB) | CRC-16 (MSB) |
| :-- | :-- | :-- | :-- | :--- | :--- |
| 5 | 1 | 1 | 1 | 1 | 1 |

**필드 설명:**

| 필드 | 길이 (Byte) | 설명 |
| :--- | :--- | :--- |
| `Len` | 1 | `0x05`. |
| `Adr` | 1 | 리더기 주소 (0x00 ~ 0xFE). 0xFF는 브로드캐스트. |
| `Cmd` | 1 | 명령어 코드. `0x76`으로 고정. |
| `Data[]`| 1 | 리더기 동작 모드. 아래 "Data[] 필드 포맷" 참조. |
| `CRC-16`| 2 | `Len`부터 `Data[]`까지의 CRC-16 체크섬. |

**Data[] 필드 포맷:**

| ReadMode |
| :--- |
| 1 |

**Data[] 파라미터 설명:**

| 파라미터 | 길이 (Byte) | 설명 |
| :--- | :--- | :--- |
| `ReadMode`| 1 | 리더기의 현재 동작 모드. <br> `0`: **응답 모드 (Answering mode)** - PC의 명령에만 응답하는 표준 모드. <br> `1`: **실시간 인벤토리 모드 (Real time inventory mode)** - 리더기가 자동으로 계속 인벤토리를 수행하고 결과를 전송. <br> `2`: **트리거 기반 실시간 인벤토리 모드 (Real time inventory mode with trigger)** - GPI1 입력 핀의 신호(Low)에 따라 실시간 인벤토리를 시작/중지. |

---

#### 2. 응답 프레임 (E310 → PC)

**프레임 구조:**

| Len | Adr | reCmd | Status | Data[] | CRC-16 (LSB) | CRC-16 (MSB) |
| :-- | :-- | :--- | :--- | :--- | :--- | :--- |
| 5 | 1 | 1 | 1 | 0 | 1 | 1 |

**필드 설명:**

| 필드 | 길이 (Byte) | 설명 |
| :--- | :--- | :--- |
| `Len` | 1 | `0x05`. |
| `Adr` | 1 | 리더기 주소. |
| `reCmd` | 1 | 응답하는 명령어 코드. `0x76`으로 고정. |
| `Status`| 1 | 명령어 실행 상태. 성공 시 `0x00`. |
| `Data[]`| 0 | 데이터 없음. |
| `CRC-16`| 2 | `Len`부터 `Status`까지의 CRC-16 체크섬. |
