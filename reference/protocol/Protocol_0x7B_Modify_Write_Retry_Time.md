### 0x7b Modify or Load Maximum Write Retry Time Command

이 명령어는 태그 쓰기 관련 동작이 실패했을 경우, 최대 몇 번까지 재시도할 것인지를 설정하거나 현재 설정된 값을 불러오는 데 사용됩니다. 기본 재시도 횟수는 3회입니다.

#### 1. 요청 프레임 (PC → E310)

**프레임 구조:**

| Len | Adr | Cmd | Data[] | CRC-16 (LSB) | CRC-16 (MSB) |
| :-- | :-- | :-- | :-- | :--- | :--- |
| 5 | 1 | 1 | 1 | 1 | 1 |

**필드 설명:**

| 필드 | 길이 (Byte) | 설명 |
| :--- | :--- | :--- |
| `Len` | 1 | `0x05`. |
| `Adr` | 1 | 리더기 주소 (0x00 ~ 0xFE). 0xFF는 브로드캐스트. |
| `Cmd` | 1 | 명령어 코드. `0x7b`로 고정. |
| `Data[]`| 1 | 쓰기 재시도 횟수 설정 값. 아래 "Data[] 필드 포맷" 참조. |
| `CRC-16`| 2 | `Len`부터 `Data[]`까지의 CRC-16 체크섬. |

**Data[] 필드 포맷:**

| RetryTimes |
| :--- |
| 1 |

**Data[] 파라미터 설명:**

| 파라미터 | 길이 (Byte) | 설명 |
| :--- | :--- | :--- |
| `RetryTimes`| 1 | **bit 7 (동작 플래그):** <br> `0`: 현재 설정된 재시도 횟수 불러오기 (이 경우 bit 0-2는 무시됨). <br> `1`: 새로운 재시도 횟수로 수정. <br> **bit 6-3:** 예약됨. <br> **bit 2-0 (재시도 횟수):** <br> `bit 7`이 `1`일 때, 설정할 최대 재시도 횟수 (0~7). |

---

#### 2. 응답 프레임 (E310 → PC)

**프레임 구조:**

| Len | Adr | reCmd | Status | Data[] | CRC-16 (LSB) | CRC-16 (MSB) |
| :-- | :-- | :--- | :--- | :--- | :--- | :--- |
| 6 | 1 | 1 | 1 | 1 | 1 | 1 |

**필드 설명:**

| 필드 | 길이 (Byte) | 설명 |
| :--- | :--- | :--- |
| `Len` | 1 | `0x06`. |
| `Adr` | 1 | 리더기 주소. |
| `reCmd` | 1 | 응답하는 명령어 코드. `0x7b`으로 고정. |
| `Status`| 1 | 명령어 실행 상태. 성공 시 `0x00`. |
| `Data[]`| 1 | 현재 설정된 쓰기 재시도 횟수. 아래 "Data[] 필드 포맷" 참조. |
| `CRC-16`| 2 | `Len`부터 `Data[]`까지의 CRC-16 체크섬. |

**응답 Data[] 필드 포맷:**

| RetryTimes |
| :--- |
| 1 |

**응답 Data[] 파라미터 설명:**

| 파라미터 | 길이 (Byte) | 설명 |
| :--- | :--- | :--- |
| `RetryTimes`| 1 | **bit 7-3:** 예약됨 (기본값 0). <br> **bit 2-0:** 현재 설정된 최대 쓰기 재시도 횟수. |
