### 0x06 Set Memory Read/Write Protection Command

이 명령어는 태그의 특정 메모리 영역(Kill 암호, Access 암호, EPC, TID, 사용자 메모리)에 대한 읽기/쓰기 보호 상태를 설정하는 데 사용됩니다.

#### 1. 요청 프레임 (PC → E310)

**프레임 구조:**

| Len | Adr | Cmd | Data[] | CRC-16 (LSB) | CRC-16 (MSB) |
| :-- | :-- | :-- | :-- | :--- | :--- |
| 가변 | 1 | 1 | 가변 | 1 | 1 |

**필드 설명:**

| 필드 | 길이 (Byte) | 설명 |
| :--- | :--- | :--- |
| `Len` | 1 | `Adr`부터 `CRC-16` MSB까지의 총 바이트 수. |
| `Adr` | 1 | 리더기 주소 (0x00 ~ 0xFE). 0xFF는 브로드캐스트. |
| `Cmd` | 1 | 명령어 코드. `0x06`으로 고정. |
| `Data[]`| 가변 | 보호 설정에 필요한 파라미터. 아래 "Data[] 필드 포맷" 참조. |
| `CRC-16`| 2 | `Len`부터 `Data[]`까지의 CRC-16 체크섬. |

**Data[] 필드 포맷:**

| ENum | EPC | Select | SetProtect | Pwd | MaskMem | MaskAdr | MaskLen | MaskData |
| :--- | :--- | :--- | :--- | :-- | :--- | :--- | :--- | :--- |
| 1 | 가변 | 1 | 1 | 4 | 1 | 2 | 1 | 가변 |

**Data[] 파라미터 설명:**

| 파라미터 | 길이 (Byte) | 설명 |
| :--- | :--- | :--- |
| `ENum` | 1 | EPC 번호의 길이를 워드(word) 단위로 지정 (0x00~0x0F). 0xFF일 경우 EPC 대신 마스크 파라미터 사용. |
| `EPC` | 가변 | 대상 태그의 EPC 번호. `ENum` 길이에 따름. |
| `Select` | 1 | 보호 상태를 변경할 메모리 영역 선택. <br> `0x00`: Kill 암호 <br> `0x01`: Access 암호 <br> `0x02`: EPC 메모리 <br> `0x03`: TID 메모리 <br> `0x04`: 사용자 메모리 |
| `SetProtect`| 1 | 설정할 보호 상태. `Select` 값에 따라 의미가 다름. <br> **(암호 영역: 0x00, 0x01)** <br> `0x00`: 보호 없음 (읽기/쓰기 가능) <br> `0x01`: 영구적 읽기/쓰기 가능 <br> `0x02`: 암호로 보호된 읽기/쓰기 <br> `0x03`: 영구적 읽기/쓰기 불가 <br> **(데이터 영역: 0x02, 0x03, 0x04)** <br> `0x00`: 보호 없음 (쓰기 가능) <br> `0x01`: 영구적 쓰기 가능 <br> `0x02`: 암호로 보호된 쓰기 <br> `0x03`: 영구적 쓰기 불가 |
| `Pwd` | 4 | 태그 접근 암호. |
| `MaskMem`| 1 | 마스크를 적용할 메모리 영역. (0x01: EPC, 0x02: TID, 0x03: User) |
| `MaskAdr`| 2 | 마스크 시작 비트 주소 (0 ~ 16383). |
| `MaskLen`| 1 | 마스크의 비트 길이. |
| `MaskData`| 가변 | 마스크 데이터. |

---

#### 2. 응답 프레임 (E310 → PC)

**프레임 구조:**

| Len | Adr | reCmd | Status | Data[] | CRC-16 (LSB) | CRC-16 (MSB) |
| :-- | :-- | :--- | :--- | :--- | :--- | :--- |
| 5 | 1 | 1 | 1 | 0 | 1 | 1 |

**필드 설명:**

| 필드 | 길이 (Byte) | 설명 |
| :--- | :--- | :--- |
| `Len` | 1 | `0x05`. |
| `Adr` | 1 | 리더기 주소. |
| `reCmd` | 1 | 응답하는 명령어 코드. `0x06`으로 고정. |
| `Status`| 1 | 명령어 실행 상태. 성공 시 `0x00`. (자세한 내용은 매뉴얼 5장 참조) |
| `Data[]`| 0 | 데이터 없음. |
| `CRC-16`| 2 | `Len`부터 `Status`까지의 CRC-16 체크섬. |
