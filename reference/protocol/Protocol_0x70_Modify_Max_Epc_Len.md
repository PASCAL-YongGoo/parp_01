### 0x70 Modify Maximum EPC/TID Length for Memory Buffer Command

이 명령어는 메모리 버퍼를 사용하는 인벤토리(`0x18`) 시, 버퍼에 저장될 태그의 EPC/TID 데이터 최대 길이를 수정하는 데 사용됩니다. 이 설정을 변경하면 이전에 버퍼에 저장된 모든 태그 데이터가 지워집니다.

#### 1. 요청 프레임 (PC → E310)

**프레임 구조:**

| Len | Adr | Cmd | Data[] | CRC-16 (LSB) | CRC-16 (MSB) |
| :-- | :-- | :-- | :-- | :--- | :--- |
| 5 | 1 | 1 | 1 | 1 | 1 |

**필드 설명:**

| 필드 | 길이 (Byte) | 설명 |
| :--- | :--- | :--- |
| `Len` | 1 | `0x05`. |
| `Adr` | 1 | 리더기 주소 (0x00 ~ 0xFE). 0xFF는 브로드캐스트. |
| `Cmd` | 1 | 명령어 코드. `0x70`으로 고정. |
| `Data[]`| 1 | 최대 길이 설정 값. 아래 "Data[] 필드 포맷" 참조. |
| `CRC-16`| 2 | `Len`부터 `Data[]`까지의 CRC-16 체크섬. |

**Data[] 필드 포맷:**

| SaveLen |
| :--- |
| 1 |

**Data[] 파라미터 설명:**

| 파라미터 | 길이 (Byte) | 설명 |
| :--- | :--- | :--- |
| `SaveLen`| 1 | 저장할 EPC/TID의 최대 길이 설정. <br> `0`: 128비트 (16 바이트). 이 경우 최대 528개 태그 저장 가능. <br> `1`: 496비트 (62 바이트). 이 경우 최대 160개 태그 저장 가능. |

---

#### 2. 응답 프레임 (E310 → PC)

**프레임 구조:**

| Len | Adr | reCmd | Status | Data[] | CRC-16 (LSB) | CRC-16 (MSB) |
| :-- | :-- | :--- | :--- | :--- | :--- | :--- |
| 5 | 1 | 1 | 1 | 0 | 1 | 1 |

**필드 설명:**

| 필드 | 길이 (Byte) | 설명 |
| :--- | :--- | :--- |
| `Len` | 1 | `0x05`. |
| `Adr` | 1 | 리더기 주소. |
| `reCmd` | 1 | 응답하는 명령어 코드. `0x70`으로 고정. |
| `Status`| 1 | 명령어 실행 상태. 성공 시 `0x00`. |
| `Data[]`| 0 | 데이터 없음. |
| `CRC-16`| 2 | `Len`부터 `Status`까지의 CRC-16 체크섬. |
